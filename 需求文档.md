# 《归园筑舍》游戏需求文档（2天开发版 v3）

> 项目名称：归园筑舍
> 引擎版本：Godot 4.3
> 目标平台：Windows
> 开发周期：2天
> 核心要求：古建筑主题、完整可通关、低门槛休闲操作

---

## 一、游戏概述

### 1.1 故事背景

离乡少年陈远回到荒废的北方四合院祖宅，在族亲陈伯和古建修复师林师傅的指引下，按照古建筑修缮的真实工序——清理院落、修复榫卯、上梁仪式、复原窗花——一步步重建古宅，传承中式古建筑匠心与乡愁文化。

### 1.2 核心玩法循环

```
清理院落（点击杂草） → 榫卯修复（选择题） → 上梁仪式（时机确认） → 窗花复原（拼图）
```

### 1.3 三个核心要求

1. **古建筑主题** — 榫卯、上梁、窗花三项传统工艺必须完整呈现
2. **2天可交付** — 砍掉一切非核心功能，只保留完整通关流程
3. **可玩可通关** — 从启动界面到结局，全流程无断裂

## 三、关卡设计

### 3.0 前置剧情·清理杂草

**场景**：主场景1（scene_main_1.tscn）

**背景**：北方四合院破旧版（灰砖灰瓦、影壁、石榴树、杂草丛生）

**NPC**：NPC1（陈伯），位于庭院入口

**流程**：
1. 玩家进入场景，与 NPC1 对话（触发 act1_meet_uncle.dialogue）
2. NPC1 指引清理庭院
3. 玩家依次点击 5 处杂草区域（Area2D，点击后隐藏）
4. 全部清除后自动触发对话（act1_clear_done.dialogue）
5. 解锁知识卡片0 → 跳转第一关

**交互**：

| 元素 | 行为 |
|------|------|
| 杂草区域 x5 | 鼠标点击 → 区域隐藏 + 播放音效 |
| NPC1 | 靠近按 F → 弹出对话 |

**实现要点**：
- 杂草用 Area2D + CollisionShape2D，点击即隐藏
- 用 GameManager.grass_cut 追踪进度
- 5处杂草全部隐藏后自动触发通关对话

---

### 3.1 第一关：榫卯修复

**场景**：level_1.tscn

**玩法**：3道选择题，看图选择正确的榫卯类型

**题库**：

| 题号 | 问题 | 选项（图片） | 正确答案 |
|------|------|-------------|----------|
| 1 | 柱类榫卯的作用是固定柱脚，下面哪类是柱形榫卯？ | 柱类/枋类/梁类图片 | 柱类（索引0） |
| 2 | 梁和柱要接在一起，得用哪种榫卯？ | 梁类/枋类/板类图片 | 梁类（索引0） |
| 3 | 枋的作用是横向拉结柱架，下面哪个是枋类榫卯？ | 枋类/柱类/梁类图片 | 枋类（索引0） |

**流程**：
1. 显示问题文字 + 3张选项图片（TextureButton）
2. 玩家点击选择
3. 正确 → 绿色提示文字 → 1.5s 后自动下一题
4. 错误 → 红色提示文字 → 可重新选择（不禁用按钮）
5. 全部答完 → 解锁卡片1 → 跳转主场景2

**实现要点**：
- 复用现有 level_1.gd 结构，扩充 questions_data 到3题
- 图片路径使用实际存在的资源
- return_scene_path 指向 main2/scene_main_2.tscn

### 3.2 第二关：上梁仪式

**场景**：level_2_beam_installation.tscn

**背景**：清理后的庭院，房屋骨架已立

**NPC**：NPC2（林师傅），在主场景2对话后进入本关

**流程**：
1. 房梁从顶部下落（AnimationPlayer 控制移动）
2. 好房梁落入目标区 → 点击"确认"按钮 → 成功
3. 坏房梁落入目标区 → 点击"确认"按钮 → 失败，重新等待下一根
4. 成功 → 解锁卡片2 → 跳转主场景3

**交互**：

| 元素 | 行为 |
|------|------|
| 房梁（好/坏） | 从顶部下落到目标区（AnimationPlayer） |
| 目标区域（Area2D） | 检测房梁是否在目标区内 |
| 确认按钮 | 点击触发判定：好梁→成功，坏梁→失败 |
| 状态标签 | 显示"等待下落"/"请确认"/"成功！"/"朽木！" |

**实现要点**：
- 用 AnimationPlayer 控制房梁移动，不用物理引擎
- 好梁/坏梁交替生成（随机顺序）
- 成功条件：好梁在目标区内时点击确认
- 失败处理：显示提示后自动生成下一根

### 3.3 第三关：窗花复原（简化版）

**场景**：window_flower_level.tscn

**玩法**：**窗花拼图** — 将散落的窗花碎片拖拽到正确位置

**流程**：
1. 显示一个窗花目标图案（参考图）
2. 下方显示 4 个打乱的窗花碎片
3. 玩家将碎片拖拽到目标区域的对应槽位
4. 全部放对 → 成功
5. 放错位置 → 碎片弹回原位
6. 成功 → 解锁卡片3 → 跳转结局

**交互**：

| 元素 | 行为 |
|------|------|
| 目标区域（4个槽位） | 接收拖拽的碎片，检测是否正确 |
| 窗花碎片（4个） | 可拖拽（TextureRect + 拖拽脚本） |
| 参考图 | 显示完整窗花样式供参考 |
| 完成提示 | 全部正确后显示成功文字 |

**实现要点**：
- 用 Control 节点的 drag & drop 实现（Godot 内置的 drag 系统）
- 4个碎片对应4个槽位，每个碎片有唯一 ID
- 拖到正确槽位 → 吸附固定；错误槽位 → 弹回原位
- 不需要绘制、不需要对称、不需要贴合度计算

> 基于现有代码结构，聚焦古建筑主题，2天可交付

---

## 一、核心原则

### 三个必须
1. **必须是古建筑主题** — 榫卯、上梁、窗花，一个不能少
2. **2天必须能做完** — 砍掉一切花哨功能，只保留核心玩法闭环
3. **必须可玩可通关** — 从启动界面到结局，完整流程跑通

### 砍掉的内容（剧情文档中删减）

| 原剧情内容 | 处理方式 | 原因 |
|-----------|---------|------|
| 交互物品探索（木马/凿子/对联/影壁） | 砍掉 | 需要额外 Area2D + 对话，时间不够 |
| 选木料环节 | 合并到上梁对话中 | 林师傅对话里一句话带过 |
| 上梁仪式焚香念诵动画 | 砍掉动画，保留文字 | 用对话展示即可 |
| 奶奶剪纸册展示画面 | 砍掉 | 需要额外 UI 插图 |
| 祖父日记结局动画 | 砍掉 | 用屏幕文字展示 |
| 关帝庙回忆支线 | 砍掉 | 支线全部砍掉 |
| 街坊来访支线 | 砍掉 | 同上 |
| 父亲电话支线 | 砍掉 | 同上 |
| 收集要素（工具/剪纸/照片） | 砍掉 | 需要额外系统 |
| 知识卡片扩展 | 保留现有4张 | 已有完整实现，不动 |

## 四、简化后的剧情流程

### 最终版流程（2天可实现）

```
启动界面（背景图 + 开始按钮）
  ↓
序幕文字（自动滚动，点击跳过）
  ↓
主场景1：破旧北方四合院
  ├─ NPC1（陈伯）对话 → 获得清理能力
  ├─ 清理5处杂草（点击交互）
  └─ 杂草清完 → 自动对话 → 跳转第一关
	↓
第一关：榫卯问答（3道选择题）
  ├─ 答对全部题目
  └─ 通关 → 解锁卡片 → 返回主场景2
	↓
主场景2：清理后的庭院
  ├─ NPC2（林师傅）对话 → 获得上梁能力
  └─ 进入上梁小游戏
	↓
第二关：上梁小游戏（房梁下落 + 时机确认）
  ├─ 好房梁落入目标区 → 点击确认
  └─ 成功 → 解锁卡片 → 跳转主场景3
	↓
主场景3：房屋完整
  ├─ NPC2 对话 → 进入窗花小游戏
  └─ 进入第三关
	↓
第三关：窗花剪纸（折叠绘制 + 对称展开）
  ├─ 折叠 → 绘制 → 展开验证
  └─ 成功 → 解锁卡片 → 跳转结局
	↓
结局：文字展示（祖父日记 + 传承字幕）
```

### 对话文件清单（只需写6个）

| 文件名 | 内容 | 角色 |
|--------|------|------|
| `prologue.dialogue` | 序幕·归乡独白 | 陈远独白 |
| `act1_meet_uncle.dialogue` | 与陈伯初遇 + 清理指引 | 陈远、陈伯 |
| `act1_clear_done.dialogue` | 杂草清完后的对话 | 陈远、陈伯 |
| `act2_meet_lin.dialogue` | 与林师傅相遇 + 上梁指引 | 陈远、林师傅 |
| `act3_beam_success.dialogue` | 上梁成功对话 | 陈远、陈伯、林师傅 |
| `act4_window_done.dialogue` | 窗花完成 + 光影满屋 | 陈远、陈伯、林师傅 |

## 六、风险与应对

| 风险 | 概率 | 应对方案 |
|------|------|---------|
| AI 生成的背景图风格不统一 | 中 | 用现有图片调色改造，不追求完美 |
| DialogueManager 插件兼容问题 | 低 | 已有测试用例，沿用现有写法 |
| 窗花小游戏绘制体验差 | 中 | 如果时间不够，改为"选择窗花图案"的简单交互 |
| 上梁小游戏物理效果异常 | 中 | 简化房梁下落逻辑，用 AnimationPlayer 替代物理 |
| 2天做不完 | 低 | 按优先级砍功能，确保核心3关可通关即可 |
